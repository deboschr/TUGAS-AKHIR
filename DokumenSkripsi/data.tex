% - abstrakENG: abstract in english 
% Petunjuk:

\abstrakINA{Keberadaan tautan rusak pada situs web dapat mengganggu aksesibilitas informasi serta menurunkan kualitas pengalaman pengguna. Untuk membantu mengatasi permasalahan tersebut, penelitian ini mengembangkan sebuah aplikasi berbasis desktop yang mampu memeriksa tautan rusak pada situs web. Aplikasi yang dikembangkan melakukan \textit{web crawling} dengan menelusuri setiap halaman pada situs web dan pada setiap halaman yang dikunjungi dilakukan ekstraksi terhadap tautan di dalamnya. Setiap tautan akan diperiksa dengan melakukan permintaan HTTP untuk menentukan apakah tautan tersebut masih dapat diakses, dan hasil pemeriksaan ditampilkan secara langsung melalui antarmuka grafis.
\\
Hasil pengujian menunjukkan bahwa aplikasi berhasil melakukan pemeriksaan tautan rusak pada situs web. Pengujian lebih lanjut mengungkap bahwa pengaturan waktu tunggu ketika menghubungi \textit{server} dan waktu tunggu untuk mendapatkan respons dari \textit{server} pada permintaan HTTP berpengaruh secara langsung terhadap keakuratan hasil pemeriksaan. Waktu tunggu yang terlalu singkat menyebabkan lebih banyak tautan yang keliru teridentifikasi sebagai tautan rusak, sedangkan waktu tunggu yang lebih panjang menghasilkan hasil yang lebih stabil meskipun mengorbankan waktu pemeriksaan yang menjadi lebih lama.}

\abstrakENG{The presence of broken links on websites can disrupt information accessibility and reduce the quality of user experience. To help address this issue, this research develops a desktop-based application capable of checking broken links on websites. The developed application performs web crawling by traversing each page within a website, and for every visited page, hyperlinks contained in the page are extracted. Each link is then examined by sending an HTTP request to determine whether the link is still accessible, and the inspection results are displayed directly through a graphical user interface.
\\
The testing results indicate that the application is able to successfully perform broken link checking on websites. Further testing reveals that the configuration of timeout values when establishing connections to the server and waiting for server responses in HTTP requests directly affects the accuracy of the inspection results. Timeout values that are too short lead to a higher number of links being incorrectly identified as broken, whereas longer timeout values produce more stable results, although at the cost of longer inspection duration.}
