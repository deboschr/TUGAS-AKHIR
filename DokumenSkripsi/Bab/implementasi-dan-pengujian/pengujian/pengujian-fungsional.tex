   Subjek pada pengujian ini adalah situs web Program Studi Informatika Universitas Katolik Parahyangan yang beralamat pada \url{https://informatika.unpar.ac.id}, hasil pengujian dapat dilihat pada Tabel~\ref{tab:hasil-pengujian-fungsional-fitur-pemeriksaan}, sedangkan ringkasan proses pemeriksaan dapat dilihat pada tabel Tabel~\ref{tab:ringkasan-proses-pemeriksaan-fitur-pemeriksaan} dan ringkasan hasil pemeriksaan dapat dilihat pada tabel Tabel~\ref{tab:ringkasan-hasil-pemeriksaan-fitur-pemeriksaan}.    \begin{itemize}

      \item Memastikan \textit{crawling} hanya dilakukan pada halaman situs web yang sedang diperiksa.

   \item \textbf{Pengujian pada Fitur \textit{Filter}}\\
   Fitur \textit{filter} terdiri dari dua kelompok kondisi, yaitu \textit{filter} berdasarkan URL dan \textit{filter} berdasarkan kode status HTTP. Kedua jenis \textit{filter} ini dapat digunakan secara terpisah maupun bersamaan sehingga pengguna dapat menyempitkan hasil pencarian sesuai kebutuhan. \textit{Filter} untuk URL mencakup empat kondisi yaitu, \texttt{Equals}, \texttt{Contains}, \texttt{Starts With}, dan \texttt{Ends With}. Sementara itu, \textit{filter} untuk kode status HTTP mencakup tiga kondisi: \texttt{Equals}, \texttt{Greater Than}, dan \texttt{Less Than}.

   Pengujian ini dilakukan untuk memastikan bahwa pengguna dapat menyaring isi tabel berdasarkan kondisi URL maupun kode status HTTP. Kondisi awal pada pengujian ini adalah proses pemeriksaan sedang atau sudah selesai menggunakan situs web Program Studi Informatika Universitas Katolik Parahyangan yang beralamat pada \url{https://informatika.unpar.ac.id}. Hasil pengujian dapat dilihat pada Tabel~\ref{tab:hasil-pengujian-fungsional-fitur-filter}.
